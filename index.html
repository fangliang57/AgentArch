05 REAL BUSINESS

真实业务场景

企业给出真实问题，
你交付可用结果

每道赛题都会说明谁在使用、目前怎样处理、企业能提供哪些资料，以及最后怎样验收。你需要把这些资料、模型和工具连成一条真正可以运行的解决流程。

  1. 01SELECT

    读懂企业问题

    找到实际使用者、当前流程，以及最耗时或最容易出错的环节。

  2. 02DEFINE

    明确完成标准

    说清楚产品要完成哪些任务、输出什么结果，以及怎样才算有效。

  3. 03BUILD

    开发并测试

    接入企业资料、模型与工具，跑通一条从输入到结果的完整核心流程。

  4. 04VERIFY

    企业现场验收

    由企业用户独立操作，用测试任务检查结果、稳定性和使用价值。

示例企业问题 / LOGISTICS

物流异常订单处置需求

在这个例子里，企业关心的是：运营人员输入一条异常订单后，能否更快获得有依据的处置建议和工单草稿。

01 / CURRENT PROBLEM

当前问题

订单、运输节点、客户规则和历史案例分散在多处，人工核对耗时，处置标准也容易产生差异。

02 / COMPANY INPUT

企业可提供

脱敏订单、运输状态、客户服务规则、历史处置案例与运营说明。

03 / EXPECTED RESULT

期待结果

输入订单后，获得异常分类、判断依据、处置建议和工单草稿；高风险或资料缺失时提示人工接管。

一次真实任务会这样运行
  1. 01输入一条异常订单
  2. 02汇总运输状态与客户规则
  3. 03判断异常原因并给出依据
  4. 04生成处置建议与工单草稿
  5. 05高风险任务提示人工接管
06 HOW TO TEAM UP

如何组队

选择适合你的
协作方式

赛事面向合作高校在校生，本科、硕士、博士均可。报名时选择一种参赛模式，并以同一项目完成后续赛程。

1 人独立参赛

一个人完成问题理解、产品设计、开发、测试和最终交付，产品方向与推进节奏由你统一掌握。

02TEAM / COLLAB

3 人组队参赛

三位成员可以跨专业互补，并围绕业务、产品、开发与测试灵活分工，共同完成同一个项目。

组队规则

赛事仅设置以上两种参赛形式；报名后，项目核心成员原则上保持固定。
